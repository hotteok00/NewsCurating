# NewsCurating - 뉴스 분석 지침

이 프로젝트는 다양한 소스에서 수집한 기사를 분석하여 주제별 종합 보고서를 작성합니다.

## 보고서 작성 규칙

1. **언어**: 모든 분석과 서술은 한국어로 작성
2. **원본 보존**: 기사 제목은 원문 그대로 유지 (영문이면 영문으로)
3. **종합 서술**: 기사를 개별 나열하지 않고, 같은 토픽의 여러 소스를 종합하여 하나의 분석 글로 작성
4. **인용**: 구체적 사실/수치를 언급할 때 `[N]` 형태로 출처를 표기

## 입력 데이터 형식

전처리된 JSON 파일은 다음 구조입니다:

```json
{
  "topics": [
    {
      "topic": "토픽 라벨",
      "ref_ids": [1, 3, 7],
      "summary": "토픽 한줄 설명"
    }
  ],
  "articles": [
    {
      "ref_id": 1,
      "title": "기사 제목",
      "content": "본문 요약",
      "key_facts": ["$110 유가", "5000명 파병"],
      "url": "https://...",
      "date": "2026-03-21",
      "source_type": "news|youtube|paper|forum",
      "content_quality": "high|medium|low",
      "source_group": "Trend|Paper|유튜브"
    }
  ]
}
```

- `topics`: LLM이 사전 클러스터링한 토픽 목록. 이를 **참고**하되 최종 토픽 구성은 자유롭게 판단
- `ref_id`: 각 기사의 인용 번호. 보고서에서 `[1]`, `[2]` 등으로 인용할 때 사용
- `related`, `related_titles`: 같은 주제의 관련 기사 (있을 경우)

## 인용 규칙

- 구체적 사실, 수치, 주장을 서술할 때 반드시 `[N]` 형태로 출처를 표기
- 같은 사실에 여러 소스가 있으면 `[1][3]` 병기
- 각 토픽 끝에 해당 토픽에서 인용한 소스를 `> **참고 소스**` 블록으로 정리
- 보고서 맨 끝 부록에 전체 참고 소스 목록을 테이블로 정리
- `ref_id`는 입력 데이터에 이미 부여되어 있으므로 그대로 사용

## 토픽 구성 규칙

- 입력 데이터의 `topics` 배열을 **참고**하되, 최종 토픽은 자유롭게 재구성 가능
- 토픽당 최소 2개 이상의 소스를 종합해야 함
- 토픽은 3~6개가 적절 (너무 많으면 "간추린 뉴스"로 이동)
- 같은 기사가 여러 토픽에 걸칠 수 있음
- 토픽에 포함된 기사는 "간추린 뉴스"에서 중복 나열하지 않을 것

## 서술 톤

- 뉴스 에이전시가 아닌, **기술/AI/로봇 분야에 관심 있는 개인**을 위한 큐레이션
- 기사를 개별 나열하지 말고, 여러 소스의 정보를 **종합하여 하나의 이야기**로 엮을 것
- 시사점은 "왜 이게 나한테 중요한지"를 기준으로 작성
- "중요하다", "주목할 만하다" 같은 피상적 표현 대신 구체적 이유와 수치를 제시
- `key_facts`에 있는 수치를 반드시 본문에 녹여서 사용

## 품질 기준

- `content`가 200자 이상이고 `content_quality`가 "high"인 기사를 우선 활용
- `content_quality`가 "low"인 기사는 참고만 하거나 생략
- `source_type`이 "youtube"이고 `content_quality`가 "low"이면 부록 "📺 영상 확인 필요"로 배치
- 광고성/홍보성 내용은 제외
